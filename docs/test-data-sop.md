# 測試資料建立 SOP

從零開始，透過 UI 手動建立出租物件、出售物件與租屋需求的完整流程。
每個步驟都附上範例資料，可直接照填。

---

## 前置準備

| 項目 | 說明 |
|------|------|
| 前端 | `http://localhost:5173` |
| 後端 | `http://localhost:8081` |
| 資料庫 | PostgreSQL（`infra/docker-compose.yml`） |

**啟動服務：**
```bash
# 資料庫
cd infra && docker compose up -d && cd ..

# 後端
cd go-service && docker compose up --build -d && cd ..

# 前端
cd react-service && npm run dev
```

**或使用 SQL 直接注入：**
```bash
# 一次注入所有測試資料（不需跑 UI 流程）
psql -U postgres -d TASK -f infra/init/14-demo-tenant-requirements.sql
psql -U postgres -d TASK -f infra/init/15-demo-properties.sql
```

## 流程一：建立屋主帳號並上架物件

### Step 1 — 建立帳號

前往 `http://localhost:5173/login`

選擇「密碼登入」，輸入：

| 欄位 | 值 |
|------|----|
| Email | `owner@test.com` |
| 密碼 | `Test1234!` |

若尚無帳號，選擇「建立帳號」後以相同資料完成註冊。

---

### Step 2 — 完成 KYC 身分驗證

前往 `http://localhost:5173/kyc`

1. **上傳身分證正面** — 使用任何測試圖片（JPG/PNG），若有真實身分證圖片最佳
2. **上傳身分證背面** — 同上
3. **上傳自拍照** — 本人正面清楚照片
4. 點選「送出審核」
5. 等待系統審核通過（開發環境通常自動或由管理員後台審核）

> KYC 通過後，`/member` 身分工作台將顯示「已驗證」狀態。

---

### Step 3 — 申請屋主憑證 (OWNER NFT)

前往 `http://localhost:5173/credential/owner`

填入申請資料（開發環境測試用）：

| 欄位 | 值 |
|------|----|
| 備註 / 申請說明 | `測試屋主帳號，用於 demo 出租出售功能` |

點選「送出申請」。

> **注意：** 開發環境需由管理員後台審核通過，才能看到 OWNER 身分工作台功能。  
> 若需快速驗證，可直接用 SQL 注入（見本文最末的快速注入章節）。

---

### Step 4 — 建立物件（出租用）

前往 `http://localhost:5173/my/properties` → 點選「新增物件」

**Step 4-1：基本建立**

| 欄位 | 值 |
|------|----|
| 物件標題 | `大安路精裝兩房採光宅` |
| 地址 | `台北市大安區大安路一段168號` |

點選「建立」。

**Step 4-2：填入詳細資料**

進入物件編輯頁，填入：

| 欄位 | 值 |
|------|----|
| 建物類型 | 大樓（BUILDING） |
| 樓層 | 6 |
| 總樓層 | 12 |
| 主建物坪數 | 21.5 |
| 房數 | 2 |
| 廳數 | 1 |
| 衛浴數 | 1 |
| 停車類型 | 無停車位（NONE） |
| 安管類型 | 兼職安管（PARTTIME） |

點選「儲存」。

---

### Step 5 — 建立並發布出租刊登

在物件頁點選「建立出租刊登」，或前往 `/my/properties/:id/rent`

| 欄位 | 值 |
|------|----|
| 月租金（元） | `35000` |
| 押金（月數） | `2` |
| 管理費由誰負擔 | 租客（TENANT） |
| 最短租期（月） | `12` |
| 開放期間（天） | `90` |
| 允許養寵物 | 是 |
| 允許開伙 | 是 |
| 備註 | `近捷運科技大樓站，採光明亮，可開伙與設籍。` |

點選「建立」後，再點「發布刊登」即完成。

> 發布後可在 `http://localhost:5173/rent` 看到此筆出租物件。

---

### Step 6 — 建立物件（出售用）

重複 Step 4，建立第二個物件：

詳細資料：

| 欄位 | 值 |
|------|----|
| 建物類型 | 大樓（BUILDING） |
| 樓層 | 8 |
| 總樓層 | 16 |
| 主建物坪數 | 42.0 |
| 房數 | 3 |
| 廳數 | 2 |
| 衛浴數 | 2 |
| 角間 | 是 |
| 停車類型 | 機械停車（MECHANICAL） |
| 安管類型 | 全天安管（FULLTIME） |

---

### Step 7 — 建立並發布出售刊登

在物件頁點選「建立出售刊登」，或前往 `/my/properties/:id/sale`

| 欄位 | 值 |
|------|----|
| 總價（元） | `45000000` |
| 每坪單價（元） | `1071429` |
| 刊登期間（天） | `90` |
| 備註 | `屋況極佳，含裝潢，近大安森林公園，全棟安管。` |

點選「建立」後，再點「發布刊登」即完成。

> 發布後可在 `http://localhost:5173/sale` 看到此筆出售物件。

## 流程二：建立租客帳號與租屋需求

> 可用同一帳號（先擁有 OWNER，再申請 TENANT），或使用另一組帳號。

### Step 8 — 申請租客憑證 (TENANT NFT)

確認已完成 KYC（Step 2），前往 `http://localhost:5173/credential/tenant`

| 欄位 | 值 |
|------|----|
| 職業類型 | 上班族 |
| 任職單位 | 科技公司 |
| 月收入區間 | 60,000 以上 |
| 同住人數（含本人） | 2 |
| 同住說明 | `伴侶同住，無小孩。` |
| 搬入時程 | 一個月內 |
| 補充說明 | `作息穩定，偏好安靜社區。` |

點選「送出申請」，等待審核通過。

---

### Step 9 — 建立租屋需求

前往 `http://localhost:5173/my/requirements` → 點選「新增需求」

**需求 1（台北大安/信義兩房）：**

| 欄位 | 值 |
|------|----|
| 目標行政區 | 台北市 大安區、信義區 |
| 最低預算（月租） | `25000` |
| 最高預算（月租） | `36000` |
| 格局備註 | `希望兩房以上，有對外窗與陽台。` |
| 預計搬入日 | 一個月後 |
| 需要寵物友善 | 是 |
| 需要停車位 | 否 |
| 最小坪數 | `12` |
| 最大坪數 | `28` |
| 最低房數 | `2` |
| 最短租期（月） | `12` |
| 可開伙 | 是 |
| 可設籍 | 是 |
| 生活風格 | `兩人同住，作息穩定，不抽菸。` |
| 必備條件 | `近捷運，可開伙，可設籍。` |

**需求 2（台北中山套房）：**

| 欄位 | 值 |
|------|----|
| 目標行政區 | 台北市 中山區 |
| 最低預算 | `16000` |
| 最高預算 | `22000` |
| 格局備註 | `單人租住，接受小坪數套房。` |
| 預計搬入日 | 一個半月後 |
| 需要寵物友善 | 否 |
| 需要停車位 | 否 |
| 最小坪數 | `8` |
| 最大坪數 | `16` |

## 快速注入（推薦給開發測試）

若不需要走完整 UI 流程，可使用以下 SQL 直接注入：

### 注入 20 筆物件（10 出租 + 10 出售）
```bash
psql -U postgres -d TASK -f infra/init/15-demo-properties.sql
```

產生內容：
- Demo 屋主帳號（wallet: `0xDemoPropertyOwner0000000000000000000000000000001`）
- 10 筆出租物件（台北市/新北市/台中市，月租 16,000–52,000 元）
- 10 筆出售物件（台北市/新北市/台中市/新竹，總價 1,980–6,800 萬元）
- 所有刊登狀態皆為 `ACTIVE`，立即可在 `/rent` 和 `/sale` 瀏覽

### 注入 5 筆租屋需求
```bash
psql -U postgres -d TASK -f infra/init/14-demo-tenant-requirements.sql
```

產生內容：
- Demo 租客帳號（wallet: `0xDemoTenant000000000000000000000000000000000001`）
- 5 筆租屋需求（台北大安/信義、中山、板橋、內湖、台中西屯/北屯）

### 一次注入全部
```bash
psql -U postgres -d TASK \
  -f infra/init/14-demo-tenant-requirements.sql \
  -f infra/init/15-demo-properties.sql
```

## 各頁面驗證

| 頁面 | URL | 預期內容 |
|------|-----|----------|
| 出租列表 | `/rent` | 10 筆出租物件，可用行政區/月租篩選 |
| 出售列表 | `/sale` | 10 筆出售物件，可用行政區/總價篩選 |
| 租屋需求 | `/requirements` | 5 筆需求（需 OWNER 或 AGENT 身份登入） |
| 我的物件 | `/my/properties` | 屋主帳號登入後可見已建立物件 |
| 我的需求 | `/my/requirements` | 租客帳號登入後可見已建立需求 |

## 搜尋功能測試案例

### 出租頁 `/rent`

| 測試 | 篩選條件 | 預期結果 |
|------|----------|----------|
| 行政區篩選 | 大安區 | 只顯示地址含「大安」的物件 |
| 月租上限 | 最高 25,000 | 顯示月租 ≤ 25,000 的物件（套房、台中物件） |
| 關鍵字 | `捷運` | 顯示 notes 或地址含「捷運」的物件 |
| 清空條件 | 點「清除」| 恢復顯示全部 10 筆 |

### 出售頁 `/sale`

| 測試 | 篩選條件 | 預期結果 |
|------|----------|----------|
| 行政區篩選 | 竹北市 | 只顯示竹北物件 |
| 總價上限 | 最高 30,000,000 | 顯示總價 ≤ 3,000 萬的物件 |
| 關鍵字 | `透天` | 顯示標題含「透天」的物件（北投、南屯） |
