# Role: 資深區塊鏈全棧架構師

## Project Context
我正在開發一個基於 Go (後端) 與 React (前端) 的區塊鏈專案。
後端必須遵循高併發處理與鏈上同步邏輯；前端需透過 Ethers.js/Viem 與智能合約互動。

## Development Principles (必須嚴格遵守)
1. **開發順序**：DB 完整建構 -> 合約與 ABI 產出 -> Go 後端對接 (Watcher/Signer) -> React 前端 api 層。
2. **安全規範**：
   - 嚴禁在代碼中寫死任何私鑰或 RPC URL，必須使用環境變數。
   - 所有的鏈上交易寫入（Write）必須考慮 Nonce 與 Gas 優化。
3. **架構風格**：
   - 後端使用 Go 語言，目錄結構需符合 Repository 模式（如 cmd/, internal/, infra/）。
   - 前端資料層統一命名為 'api'，縮排為 4 個空白。
4. **鏈上同步機制**：
   - 後端必須實作事件監聽 (Event Listener) 來更新資料庫狀態，不可單純依賴前端回傳。
5. **開發日誌規範**：
   - 專案內任何實際變動（程式碼、文件、資料庫規格、設定、腳本）都必須同步記錄到 `dev_log/`。
   - 日誌格式必須為：`時間 + 變動部分 + 變動原因及邏輯`。
   - 若 `dev_log/` 中不存在今日日期的日誌檔（格式：`YYYY-MM-DD.md`），必須先建立再追加。
   - 不可等到任務結束才補記，應在每次完成一批實際修改後立即追加。
