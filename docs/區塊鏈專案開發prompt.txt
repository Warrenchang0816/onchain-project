## Development Principles (必須嚴格遵守)
1. **先判定 Gate**：
   - 先確認這個需求屬於 Gate 1、2、3、4 還是 5。
   - 不可把下一個 Gate 的內容誤寫成已經進行中的正式主線。

2. **開發順序**：
   - DB / read model
   - 合約事件與函式介面
   - Go service / indexer / signer
   - React `api` / page

3. **安全規範**：
   - 嚴禁在程式碼中寫死任何私鑰、RPC URL、SMTP、雲端憑證。
   - 前端 `.env` 視同公開資訊，不可放 secret。
   - 所有鏈上寫入的正式狀態都要由 event / indexer / read model 驗證，不可只依賴前端回傳。

4. **架構風格**：
   - Go 後端遵守 `cmd/`、`internal/`、`infra/` 的分層。
   - `platform/*` 只放基礎能力，商業邏輯放在 `modules/*`。
   - React 資料層統一命名為 `api`。

5. **鏈上同步機制**：
   - 後端必須有事件監聽或可重播同步策略，不可把鏈上狀態更新責任交給前端。
   - 若功能尚未有 indexer / read model 閉環，要明確標示為未完成 Gate。

6. **文件與日誌規範**：
   - 任何實際變動（程式碼、文件、資料庫規格、設定、腳本）都必須同步記錄到 `dev_log/YYYY-MM-DD.md`
   - 格式：`時間 + 變動部分 + 變動原因及邏輯`
   - 若涉及主幹方向，還要同步檢查 `docs/開發規劃書.md`、`docs/架構設計書.md` 是否需要更新
