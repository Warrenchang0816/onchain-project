## 目前使用方式

- 團隊開發應以 PR 為主，等待 `Go CI`、`Frontend CI`、`Contracts CI`、`Security` 完成後再 review / merge。
- 後續建議於 GitHub `Branches` 設定中，將上述四條 workflow 設為 `main` 的 required checks。

## 目前限制

- dependency review 的正式入口已存在，且 repo variable `ENABLE_DEPENDENCY_REVIEW=true` 可維持開啟。
- 但 GitHub 平台對此 repo 目前仍可能回 `unsupported / forbidden`，因此 workflow 採取預檢 + graceful skip 的策略，不把平台側狀態誤判成程式碼問題。
